## Official Form Requirements

The OpenAI form asks for:

- public GitHub username;
- public GitHub repository URL;
- maintainer role;
- why the repository qualifies;
- interest in Codex Security and/or API credits;
- OpenAI organization ID;
- how API credits will be used;
- optional additional notes.

## Suggested Answers

### Role

I am the primary maintainer and creator of Money Engine AI. I lead product direction, architecture, implementation, documentation, releases, issue triage, security policy, and review of community-contributed revenue playbooks.

### Why This Repository Qualifies

Money Engine AI is a self-hosted OSS revenue experimentation tool for founders and small businesses. It turns profiles into scored revenue paths, offers, 7-day sprints, metrics, reviews, and reusable community playbooks, with ethical no-spam/no-guarantees rules.

### API Credit Usage

We will use API credits for OSS maintainer workflows: validating structured AI outputs, prompt/eval fixtures, playbook regression tests, PR review, issue triage, docs, release notes, example generation, and safe automation for the project's core AI workflows.

### Additional Notes

I am dogfooding the project publicly and will maintain issues, releases, roadmap, security policy, examples, and community playbook contributions. The goal is a real self-hosted OSS alternative to closed AI business/revenue coaching tools.
